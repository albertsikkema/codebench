# Branching Rules

## Branch Naming

- `feat/description` — new features
- `fix/description` — bug fixes
- `refactor/description` — code restructuring
- `chore/description` — maintenance tasks

Use short, lowercase, hyphenated descriptions. Include the task ID when one exists.

Examples: `feat/user-auth`, `fix/login-redirect`, `feat/T-012-add-caching`.

## Workflow

- Never commit directly to main
- Create a feature branch before making changes
- Check current branch before committing — if on main, create a branch first
- Do not push without explicit user approval
